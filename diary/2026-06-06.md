## What was done

- Verified the backend boots cleanly — uvicorn starts, `GET /health` returns `200 {"status":"ok"}` — using the existing venv.
- Shipped Phase 9 (Monitoring & Evaluation): an `llm_logs` table plus a logging layer that records one row per Claude/embedding call (model, tokens, latency, endpoint) from inside the `llm/` layer, so no service signatures had to change.
- Centralised every Claude call through a single `anthropic_client.complete()` chokepoint — quiz generation, short-answer grading, and plan generation no longer spin up their own Anthropic clients.
- Wired Sentry on both backend (behind `SENTRY_DSN`) and frontend (`@sentry/nextjs`, gated on `NEXT_PUBLIC_SENTRY_DSN`), both no-ops when the DSN is empty.
- Added `GET /admin/metrics` (DAU, quiz completions, uploads, total tokens) gated by an `ADMIN_EMAILS` allow-list, plus a `docs/eval/` rubric + runner + starter sample set for the companion.
- Shipped Phase 10 (Deployment): GitHub Actions CI (backend ruff+pytest, frontend lint+build), a Render blueprint (`render.yaml`), a production `docker-compose.prod.yml`, `DEPLOYMENT.md`, and an end-to-end `docs/smoke-test.md`. **MVP is now feature-complete across Phases 0–10.**

## Problems encountered

- The database connection failed everywhere: the Supabase pooler rejected our credentials with "tenant/user not found". The project ref resolves and the host is reachable, but auth is refused — so all DB-dependent verification was blocked.
- The first CI run went red: `npm ci` failed on the Linux runner, claiming `package.json` and `package-lock.json` were out of sync (missing `@emnapi/*`).

## Solutions

- Traced the Supabase failure to a **paused project** (free-tier projects auto-pause after inactivity, and the pooler returns exactly that error). Documented it and deferred the DB-dependent checks rather than guessing at credentials.
- Root-caused the CI failure: the lockfile was generated on Windows and omits Linux-only optional deps, which strict `npm ci` rejects. Switched CI to `npm install` to match the frontend Dockerfile. Re-run went green — backend and frontend both pass, and the deploy workflow no-ops cleanly when no Render hook secret is set.
- Built the LLM logger to open its own short-lived session and swallow all errors, so monitoring can never break a user request.
