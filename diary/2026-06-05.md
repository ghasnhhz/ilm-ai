## What was done

- Shipped the AI learning companion: grounded chat over uploaded materials with citations, in Uzbek/Russian/English.
- Shipped quiz mode: generates multiple choice and short answer questions from the user's materials, grades them, and saves scores.
- Switched embeddings from OpenAI to local sentence-transformers (all-MiniLM-L6-v2, 384 dimensions) to drop the external dependency and run fully offline.
- Built knowledge gap detection: a live "Gaps Report" computed from quiz history. A concept counts as a gap when it's answered wrong at least twice across at least two sessions; strong concepts are the ones above 80% accuracy.
- Added the `GET /gaps` endpoint and a `/gaps` page with a strong-vs-weak breakdown and links back to the materials worth revisiting.

## Problems encountered

- TODO.md had drifted out of sync — Phases 3 and 4 were finished in the code but still marked as not started, so it was no longer a reliable picture of progress.
- The brief specified the gaps endpoint as `GET /gaps/{user_id}`, which would let any logged-in user read another person's report just by changing the ID in the URL.

## Solutions

- Re-synced TODO.md against the actual code and adopted a rule: a phase is only marked complete once it's merged. Each phase now gets its own feature branch, commit, push, and merge.
- Scoped the gaps endpoint to the current user from their JWT instead of a path parameter. The underlying `compute_gaps(db, user_id)` function stays reusable, so the Phase 6 learning-plan agent can call it directly as a tool.
