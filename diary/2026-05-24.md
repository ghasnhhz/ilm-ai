## What was done

- Built the file upload endpoint: accepts PDFs and text documents.
- Implemented the document processing pipeline: chunking, embedding, storing in pgvector.
- Embeddings use sentence-transformers locally (all-MiniLM-L6-v2, 384 dimensions).
- Got basic RAG chat working — the AI answers questions grounded in uploaded documents.
- Implemented Quiz mode: generates multiple choice and short answer questions from uploaded material.

## Problems encountered

- Installed Docker Desktop to containerize the app but my laptop ran out of disk space completely, which blocked all further work until resolved.
- Claude Code accidentally mixed code from two feature branches — it had not pushed the previous branch before starting work on the next one, which caused conflicts.

## Solutions

- Uninstalled Docker Desktop (only viable option given disk space). Rewrote the setup to run FastAPI, PostgreSQL, and the frontend locally without Docker for now — will revisit containerization in Week 4.
- Fixed the branch issue by giving Claude Code an explicit prompt describing exactly what it had done wrong. It corrected the mistake cleanly. Learned an important lesson: when using Claude Code across multiple feature branches, always explicitly confirm the previous branch is committed and pushed before starting the next task.
