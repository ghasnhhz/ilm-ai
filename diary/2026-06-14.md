## What was done

- Fixed Telegram account linking, which was completely broken.
- Fixed the Telegram bot being totally unresponsive — it now replies to /start and commands.
- Verified the whole flow live: backend + frontend + bot running locally, link a chat, run a quiz, set a reminder.

## Problems encountered

- Clicking "Start" in the bot did nothing at all — no reply.
- Account linking never completed: pressing the "Connect Telegram" deep link just opened a generic welcome and never linked the account.

## Solutions

- Linking: the deep link was carrying a long JWT, but Telegram's start parameter only allows short tokens (≤64 chars, limited characters), so it was silently dropped. Replaced the JWT with a short single-use token that expires after 10 minutes.
- Unresponsive bot: the bot process never loaded the `.env` file, so it started with no Telegram token and crashed immediately — nothing was polling. Made the bot load `.env` the same way the backend does. Now it starts cleanly and responds.
